The World Wide Web

The web is composed largely of HTML documents.
HTML stands for HyperText Markup Language. HTML is the language the browsers use to interpret web pages. HTML defines the structure of a page. DOM reads the HTML to define the tree, and CSS (Cascading Style Sheets) imposes style.
The client and servers interact through something called the Hypertext Transfer Protocol (HTTP)

The components of the web include
+ the client (your computer and browser)
+ the Internet
+ the servers

Basic HTML Elements

HTML elements are what we use to tell a web browser how to display content on a web page. Most HTML elements consist of an opening and a closing tag with some content between them. Elements can also have attributes (see below), which are used by CSS.

a basic HTML element
<b>tag attribute content<b/>

<b>text<b/> Makes the 'text' content bold
<em>text<em/> Makes the 'text' content italic
<p>text<p> Defines a paragraph. The content of the <p> element is in a "box"
<br> Creates a line break
<div>content</div> Creates a "box" container element. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
<span>content</span> Creates an "inline" container element

HTML Attributes

Some elements accept additional values called attributes in their opening tags. Attributes provide additional information to the browser. For example, the anchor tag takes an attribute called href that defines a URL to link to. Attributes also provide information to CSS to enable styling.
<a href="URL>link</a> Creates a hyperlink to the href attribute value url and names it link

<img src="URL" alt="text> Displays the image hosted at the src attribute value "url". In this example, "img" is the tag and "src" is an attribute. Note that images don't have closing tags (void tag)

How HTML becomes the Web

HTML tags are read by Document Object Model (DOM) to form elements in a tree. For any web page, you can use Chrome Developer Tools ("inspect element") to see the tags and trees.
HTML is read by DOM. CSS converts HTML attributes into styled pages.

Cascading Style Sheets (CSS)

+ You can have multiple CSS for one HTML document.

CSS vocabulary
Rule: a line of CSS code describing the value that a certain attribute should take. The most specific rule applies (everywhere).
Property: The property you want to change.
Value: The value that you want to assign to the attribute.
Selector: The name that you use to in order to target the elements that are assigned to a class or id attribute in the HTML.
Class: A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks such as class="1.1"
ID: ID's are unique identifiers that uniquely identifies an element in HTML.
CSS formatting
.class name from HTML (aka CSS "selector") { CSS attribute: value }

Where to put your CSS
Preferred method: Link your HTML to a separate CSS file (best idea)

  • Write all of your structural HTML in one file (let's call it main.html).
  • In a separate file (let's call it main.css), put all of your CSS code.
  • Add a element to the top of your HTML.
  • Add a tag inside the element. Since is a "void tag" you don't need to add a closing .
  • Add the following attributes to your :
    • rel="stylesheet"
    • href="main.css"

Box Sizing and Positioning in CSS

HTML elements are boxes and each box has 4 components.
Because there are so many components to each box, it can often be hard to get the size of a box just right. There are two techniques you can use to help deal with sizing issues:

  • Set sizes in terms of percentages rather than pixels.
  • Set the box-sizing attribute to border-box for every element using * {
  • Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
